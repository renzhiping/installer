# 合同审核插件使用指南

## 工具：`contract_review`

统一工具入口，支持三个操作。

### 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `action` | string | ✅ | 操作类型：`forward` / `status`  |
| `user_id` | string | ✅ | 用户 ID,请传递这次聊天的真实用户id,没有则填null，
| `platform` | string | ✅ | 平台，如：`feishu` `weixin` |
| `text` | string | 仅 forward | 审核意图文本 |
| `task_id` | string | 该参数会自动填充,几乎没有场景需要我主动使用此参数,除非用户明确要对之前任务做操作才传递此参数,否则不传 |
| `files` | array | 仅 forward | 文件列表，每项 `{filename, path}` |

## 工作流程

### 1. forward（提交审核）

提交审核,补充信息取消任务等，返回 `task_id` 。

### 2. status（查询状态）

需要 `task_id` 参数。返回任务当前状态和阶段。

## 注意事项
- 若用户已经明确表达了审核意图,我不要再对意图做任何修改直接传递给插件
- 注意这是一个远端服务非同步插件,我可能不会收到过程和结果,若我没收到说明插件直接推送给用户
- 下次用户发送消息时,插件会自动将之前推送给用户的消息发送给我,这些消息用户都已经看到过了,我可以根据这些消息继续调用工具,无需再复述消息内容 
- 有时插件会要求用户补充信息，用户会发给我，我直接调 `contract_review(action='forward', ...)` 补充即可,后续状态信息会自动注入到上下文,我发送请求后无需主动再次查询插件状态
- 运行流程有两种情况：
  - 已登录：直接调用，上传文件，提交任务，返回任务id,后台推送进度通知给用户
  - 未登录：调用后我收到等待登陆,用户会收到登录链接，用户登录后插件会自动重放请求，无需我二次发送,后台推送进度通知给用户
